объектно-ориентированный подход и реализация на C++"

СОДЕРЖАНИЕ

1. Структура компакт-диска

1. Структура компакт-диска.
   -----------------------

Компакт-диск имеет следующую структуру папок и файлов:

Readme.txt                - настоящий файл

DJGPPInstall\             - файлы, необходимые для установки компилятора и
                            системы программирования DJGPP фирмы Delorie Software
   readme.1st             - инструкция по установке и использованию DJGPP 
                                     (на английском языке)
   copying                - лицензия GNU на использование системы DJGPP
   GPL-Rus.txt            - перевод Стандартной Общественной Лицензии GNU 
                                      на русский язык
   copying.dj             - лицензия GNU на использование продукции DJ Group
   copying.lib            - лицензия GNU на использование библиотеки GPL
   unzip32.exe            - программа распаковки .zip-архивов

   djdev203.zip           - инструменты разработчика и программы поддержки
                            периода исполнения (DJGPP версии 2)
   faq230b.zip            - список ответов на часто задаваемые вопросы (FAQ) в
                            различных форматах (Info, ASCII, HTML...)
   rhide15b.zip           - интерактивная среда разработки программ
   bnu214b.zip            - утилиты фильтрации, поиска строк и др.
   gcc333b.zip            - компилятор языков С и С++
   gdb53b.zip             - отладчик программ, компилированных системой DJGPP
   gpp333b.zip            - библиотеки С++ (STL и др.)
   mak3791b.zip           - make - программа поддержки и сборки проектов
   txi46b.zip             - info - подсистема помощи
   unzipAll.bat           - пакетный файл для распаковки всех архивов

Programs\                 - файлы, содержащие тексты примеров программ, 
                                     приводимых в книге
   Chapter1\              - примеры главы 1
   Chapter2\              - примеры главы 2
   Chapter3\              - примеры главы 3
   Chapter4\              - примеры главы 4
   Chapter5\              - примеры главы 5
   Chapter6\              - примеры главы 6
   Chapter7\              - примеры главы 7
   Chapter8\              - примеры главы 8

Все примеры, содержащиеся на диске, полностью готовы к исполнению под управлением
любой из операционных систем фирмы Microsoft, способной к запуску программ,
написанных для операционной среды MS-DOS. См. раздел 3 настоящего текста для
получения инструкций для запуска и использования примеров.

2. Установка компилятора и системы программирования на С++ фирмы Delorie
   Software.
   -------------------------------------------------------------------------------

Для установки системы программирования DJGPP на Вашем компьютере Вам необходимо
выполнить следующие действия:

- создайте на жестком диске Вашего компьютера каталог, в который будет 
  установлена система, скажем, C:\DJGPP. Вы можете также установить систему в 
  любой другой каталог, но он не должен иметь имя DEV или быть подкаталогом
  каталога с таким именем; кроме того, не используйте длинные имена файлов и
  имена, содержащие пробелы и специальные символы. Если Вы устанавливаете
  систему в другой каталог, то замените в последующей инструкции имя C:\DJGPP на
  имя выбранного Вами каталога;
- перенесите во вновь созданный каталог все или только необходимые (см. п. 3)
  файлы из каталога DJGPPInstall приложенного компакт-диска;
- распакуйте необходимые архивы с сохранением структуры подкаталогов. Для
  нормальной работы системы с примерами книги минимально необходимо распаковать
  файлы djdev203.zip, gcc333b.zip, bnu214b.zip и gpp333b.zip. Для распаковки
  можно воспользоваться приложенной бесплатной программой распаковки архивов
  unzip32.exe. Программа распаковки должна поддерживать длинные имена файлов.
  Для распаковки всех архивов можно просто запустить на исполнение пакетный файл
  unzipAll.bat;
- установите значение переменной среды DJGPP так, чтобы она указывала на файл
  C:\DJGPP\djgpp.env, кроме того, добавьте путь к каталогу C:\DJGPP\bin к
  переменной среды PATH. Если Вы не знаете, как изменить переменные среды, то
  можете прочитать об этом в инструкции Readme.1st, которая окажется в каталоге
  C:\DJGPP после распаковки архивов. Инструкция написана на английском языке;
- перед первым использованием системы Вам может понадобиться перезагрузить
  компьютер, чтобы актуализировать изменения переменных среды. Перезагрузка не
  требуется для операционных систем Windows NT, Windows 2000 и Windows XP.

Система готова к использованию.

3. Запуск и использование программ, записанных на диске.
   ----------------------------------------------------

Все программы, содержащиеся на диске, можно копировать, изменять, отлаживать и
запускать независимо от других программ. Все файлы исходных текстов, относящиеся
к одному примеру, находятся в отдельном каталоге, не имеющем подкаталогов. Так,
например, все файлы, составляющие программу работы со словарем, организованном в
виде двоичного дерева, находятся в каталоге Programs\Chapter3\3.3\TreeDictionary.
Здесь имя каталога Chapter3 означает главу 3, имя каталога 3.3 - номер раздела,
содержащего пример, TreeDictionary - название примера.

Каждый каталог содержит необходимые текстовые файлы *.h и *.cpp, в каждом примере
файл main.cpp содержит только текст, который использует определенные в примере типы
данных и алгоритмы для исполнения простых проверочных действий. Программы
используют только простой текстовый вывод данных; ввод исходных данных не
используется вовсе. Все необходимые исходные данные создаются непосредственно в
файле main.cpp.

Для запуска программ Вы можете использовать один из следующих методов. Во-первых,
Вы можете непосредственно задавать команды компиляции, сборки и выполнения
программ из командной строки DOS (для облегчения этой задачи все примеры снабжены
пакетным файлом compile.bat, который осуществляет компиляцию, сборку и запуск
примера с остановкой в конце работы).

Во-вторых, Вы можете для каждого примера определить свой .mak-файл и запускать
компиляцию, выполнение и отладку примеров с помощью утилиты make.exe.

Наконец, Вы можете воспользоваться интерактивной средой программирования rhide,
которая позволяет в интерактивном режиме компилировать, запускать и отлаживать
программы на С++. Для того чтобы пользоваться оболочкой rhide, примеры должны
лежать в каталоге, путь к которому не содержит пробелов и специальных символов в
именах файлов (в число "специальных символов" входят и русские буквы!). Конечно,
программа rhide должна быть установлена Вами - для этого Вы должны в процессе
установки системы распаковать файл rhide15b.zip.

Удобнее всего пользоваться интерактивной средой rhide следующим образом. Откройте
окно командной строки MS-DOS и сделайте текущим каталогом тот каталог, в котором
содержится исследуемый Вами пример (это можно сделать, например, с помощью
команды chdir). Затем запустите оболочку с помощью команды rhide. После этого Вы
можете открывать и редактировать исходные тексты программ (команды меню File),
компилировать и собирать готовую программу (команды меню Compile), запускать
программу с помощью команд меню Run и отлаживать ее с помощью команд меню Debug.

Конечно, если в Вашем распоряжении имеется более мощная система программирования
на С++, Вы можете загрузить примеры в Вашу систему программирования и не
использовать DJGPP вовсе. Следует только иметь в виду, что другие системы
программирования могут трактовать некоторые конструкции и средства языка по-
другому, чем это делает система DJGPP. Различия могут касаться описания и правил
видимости идентификаторов во вложенных классах, описания реализаций методов
шаблонов, инициализации массивов и т.п. Автор гарантирует, что все примеры
компилируются без ошибок компилятором DJGPP (иногда, впрочем, компилятор может
выдавать предупреждения, которые не влияют на ход выполнения программы).
